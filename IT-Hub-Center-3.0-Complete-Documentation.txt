1. PROJECT OVERVIEW
================================================================================

IT Hub Center 3.0 is an enterprise IT Service Management (ITSM) platform built
with React, TypeScript, and Supabase. It provides comprehensive management of:

- IT Support Tickets (with SLA tracking)
- Hardware/Software Assets (with depreciation & QR codes)
- Software Licenses (with seat assignments & expiration alerts)
- Operational Incidents (with timeline & AI post-mortem)
- Knowledge Base (multilingual articles with AI generation)
- Vendor Management (contracts, interactions, documents)
- Employee Onboarding/Offboarding workflows
- Reports & Analytics (PDF & Excel export)
- AI-powered Assistant (context-aware chatbot)

The platform supports 3 languages (English, Italian, Spanish), role-based access
control (4 roles), dark/light themes, and is installable as a Progressive Web App.

4. AUTHENTICATION FLOW
================================================================================

File: src/contexts/AuthContext.tsx, src/services/auth.ts

FLOW:
1. User navigates to /login
2. Enters email + password
3. auth.ts: login() calls supabase.auth.signInWithPassword()
4a. If no 2FA: Returns user, AuthContext sets state, navigate to /dashboard
4b. If 2FA enabled: Returns requires2FA flag, shows TOTP input
    User enters 6-digit code from authenticator app
    auth.ts: verify2FA() calls supabase.auth.verifyOtp()
    AuthContext sets user state, navigate to /dashboard
5. AuthContext starts activity listeners (mouse, keyboard, touch events)
6. After 1 hour of inactivity: automatic logout
7. Real-time auth state listener updates user on session changes

FEATURES:
- 10-second timeout on login/2FA to prevent hanging
- 3-second safety valve in AuthContext to prevent infinite loading
- Retry logic (3 attempts) for profile fetching after signup
- Fallback profile creation if database trigger is missing
- User status tracking (Online, Away, Do Not Disturb, Invisible)

REGISTRATION:
- Email + password + name + role
- Supabase Auth handles email verification
- Profile auto-created via database trigger (with fallback)

PASSWORD RESET:
- Sends reset email via Supabase Auth
- Redirects to /update-password with token

5. DATA LAYER & REAL-TIME ARCHITECTURE
================================================================================

File: src/contexts/DataContext.tsx, src/services/api.ts, src/utils/mappers.ts

INITIAL DATA FETCH:
On mount, DataProvider fetches ALL data in parallel:
  Promise.all([
    getTickets(), getAssets(), getLicenses(),
    getIncidents(), getKBArticles(), getUsers()
  ])

REAL-TIME SUBSCRIPTIONS:
DataContext subscribes to Supabase postgres_changes on ALL tables:
  - profiles (user changes)
  - tickets (ticket CRUD)
  - assets (asset CRUD)
  - licenses (license CRUD)
  - license_assignments (seat assignments)
  - comments (ticket comments)
  - incidents (incident CRUD)
  - kb_articles (article CRUD)

Each change triggers a state update using the mapper functions.

DATA MAPPING:
src/utils/mappers.ts normalizes Supabase rows into TypeScript types:
  - mapProfileToUser() - DB profile → User
  - mapTicketData() - DB ticket → Ticket (with user/asset lookups)
  - mapAssetData() - DB asset → Asset (with user lookup)
  - mapLicenseData() - DB license → License (with assignments)
  - mapIncidentData() - DB incident → Incident (with user lookup)
  - mapKBArticleData() - DB article → KBArticle (with author)
  - mapCommentData() - DB comment → Comment (with author)

AUTOMATION:
On DataProvider mount, if user is Admin:
  checkLicenseExpirationsAndCreateTickets() - auto-creates tickets for
  licenses nearing expiration.

DATABASE TABLES:
  - profiles - User accounts & preferences
  - tickets - Support tickets
  - comments - Ticket comments
  - assets - IT equipment inventory
  - licenses - Software licenses
  - license_assignments - License-to-user mapping
  - incidents - Operational incidents
  - kb_articles - Knowledge base articles
  - vendors - Vendor contacts
  - vendor_interactions - Vendor communication threads
  - vendor_comments - Vendor interaction messages
  - user_integrations - Third-party API keys
  - sla_policies - SLA response/resolution times

STORAGE BUCKETS:
  - avatars - User profile pictures
  - attachments - Ticket file attachments
  - assets - Asset images
  - documents - Vendor documents

7. CONTEXT PROVIDERS (STATE MANAGEMENT)
================================================================================

1. AuthContext (src/contexts/AuthContext.tsx)
   State: user, status, isAuthenticated
   Methods: login(), verify2FA(), register(), logout(), updateUser(), setUserStatus()
   Features: Inactivity timer, real-time auth state, activity tracking

2. DataContext (src/contexts/DataContext.tsx)
   State: tickets[], assets[], licenses[], incidents[], articles[], users[], isLoading
   Methods: refetchData(dataType)
   Features: Real-time subscriptions, data mapping, automation triggers

   State: language
   Methods: setLanguage(), t(key, params)
   Features: 3 languages, parametrized translations

4. NotificationsContext (src/contexts/NotificationsContext.tsx)
   Purpose: In-app toast notifications

5. PWAInstallContext (src/contexts/PWAInstallContext.tsx)
   State: installPrompt, canInstall, isInstalled
   Features: Web app install detection

Provider nesting order (src/App.tsx):
  ErrorBoundary > LocalizationProvider > AuthProvider > PWAInstallProvider >
  HashRouter > [DataProvider + NotificationsProvider (inside authenticated routes)]

10. AI INTEGRATION (CLAUDE + GEMINI)
================================================================================

CLAUDE (via Supabase Edge Function):
  File: supabase/functions/claude/index.ts
  Model: claude-sonnet-4-5
  Architecture: Deno edge function as secure proxy
  Prevents: API key exposure on client side

  Request: POST /functions/v1/claude
  {
    messages: [{role: "user"|"assistant", content: "..."}],
    system: "optional system prompt",
    max_tokens: 1024
  }

  Response:
  {
    content: [{type: "text", text: "..."}],
    model: "claude-sonnet-4-5",
    usage: {input_tokens: N, output_tokens: N}
  }

GEMINI (direct client integration):
  Model: gemini-2.5-flash
  API Key: VITE_GEMINI_API_KEY (via import.meta.env)

  Used for:
  - generateTicketReply() - Smart ticket reply suggestions
  - generatePostMortemSummary() - Incident post-mortem analysis
  - translateTextWithAI() - Text translation between languages
  - getAIWritingAssistance() - Grammar correction / text completion
  - generateArticleContentFromTopic() - KB article generation
  - getAIChatResponse() - AI chatbot responses

AI ASSISTANT COMPONENT (src/components/AIAssistant.tsx):
  - Floating chat widget (bottom-right corner)
  - Fetches context: KB articles, tickets, incidents
  - Maintains conversation history in local state

13. UTILITIES
================================================================================

   All mapper functions convert Supabase DB rows to TypeScript interfaces.
   Includes unknown user fallback for deleted/missing profiles.

2. localization.ts - Translation dictionary
   1000+ translation keys across 3 languages.
   Categories: general UI, asset statuses, asset types, ticket statuses,
   incident categories, report labels, settings labels, error messages.
   Supports parametrized strings: "Hello {name}" -> "Hello World"

3. depreciation.ts - Asset value calculation
   Straight-line depreciation model.
   10% salvage value assumption.
   Type-based lifespan (e.g., Laptop=4y, Desktop=5y, Smartphone=2y).
   Smart type inference for 'Other' category.

4. eventBus.ts - Pub/sub event system
   on(event, callback) - Subscribe
   off(event, callback) - Unsubscribe
   emit(event, ...args) - Publish

20. KEY ARCHITECTURAL PATTERNS
================================================================================

1. Context-based State Management
   No Redux. React Context API for global state. Simpler for this app size.

2. Real-time Subscriptions
   DataContext subscribes to Supabase postgres_changes. No polling needed.

3. Centralized Data Mapping
   mappers.ts prevents inconsistency between initial fetch and realtime updates.

4. Ref-based Subscription Callbacks
   Uses useRef to provide latest state without re-subscribing on every render.

5. Secured AI Proxy
   Claude calls go through Supabase Edge Function. API key never on client.

6. Straight-line Depreciation
   10% salvage value. Type-based lifespans. Smart inference for 'Other'.

7. Inactivity Auto-logout
   1 hour timeout. Resets on mouse/keyboard/touch activity.

8. HTML Sanitization
   XSS prevention via whitelist-based HTML sanitizer for user content.

9. Cryptographic Random IDs
   crypto.randomUUID() for file naming instead of Math.random().

10. Error Boundary
    Top-level React ErrorBoundary prevents full-app crashes.
